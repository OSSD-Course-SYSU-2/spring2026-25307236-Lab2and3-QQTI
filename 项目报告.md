QQTI人格分析系统 —— 项目报告


一、项目概述

项目名称：QQTI人格分析系统

开发平台：DevEco Studio（华为 HarmonyOS 官方 IDE）

目标平台：HarmonyOS（支持手机、平板、2-in-1 桌面设备）

开发语言：ArkTS（基于 TypeScript 扩展的 HarmonyOS 声明式 UI 语言）

UI 框架：ArkUI

应用包名：com.example.myapplication


二、项目背景

近年来，各类人格测试在社交网络上风靡一时，如 MBTI（Myers-Briggs Type Indicator）、SBTI 等性格测试类内容在 QQ 空间、微信朋友圈等平台被广泛传播与分享。这类测试往往以"测测你的人格类型""看看你的性格特点"为噱头，吸引用户参与互动。

然而，部分不法分子借此机会，将恶意信息收集伪装成"人格测试"页面，诱导用户输入 QQ 号、手机号、身份证号、家庭住址等个人敏感信息，存在严重的隐私泄露和网络诈骗风险。许多人出于好奇或从众心理，在未经思考的情况下轻易提交了自己的真实信息。

为了宣传防骗意识、提高个人信息安全意识，我们开发了本项目——QQTI人格分析系统。该项目以"人格测试"的外壳包装，但在用户输入 QQ 号后，并不会生成任何分析结果，而是直接播放经典反诈视频《Never Gonna Give You Up》（Rickroll），以出其不意的方式提醒用户：不要随意在网络上输入个人敏感信息，谨防上当受骗。

设计理念：用幽默和教育相结合的方式，让用户在"被整蛊"的同时深刻记住信息安全的重要性。


三、项目版本迭代

v1.0：基础功能，QQ号输入后跳转视频播放，实现反诈提醒核心流程

v1.1：一次开发多端部署，适配手机、平板、2-in-1 PC 等多种设备形态，实现响应式布局

v1.2：多元流转，支持跨设备迁移，用户在手机端输入后可无缝切换到平板继续观看；增加演示视频


四、功能设计

4.1 主页——QQ号输入页面（Index）

对应文件：pages/Index.ets

- 标题"QQTI分析系统"，使用蓝色主题色 #1A73E8，模拟专业人格测试的视觉风格
- 输入提示"请输入您的QQ号（不足10位后面补0）"，限制输入为 10 位纯数字

- 展示测试说明（"系统将根据QQ号分析您的性格特点"等），进一步增强页面可信度
- 验证通过后通过 router.pushUrl 跳转至 VideoPage，传递 QQ 号参数

4.2 视频播放页面——反诈提醒（VideoPage）

对应文件：pages/VideoPage.ets

- 内嵌 Video 组件播放反诈教育视频（《Never Gonna Give You Up》）
- 提供"播放"和"暂停"按钮，完整视频播放体验
- 安全警告卡内容：
  ⚠ "您刚刚输入了QQ号！"
  "在网络上随意输入个人信息存在安全风险"
  "请提高个人信息安全意识，谨防网络诈骗"
  "视频内容：个人信息保护与反诈骗知识"
- 警告卡使用橙黄色背景 #FFF3E0，红色字体突出安全警示

4.3 安全验证失败页面

当用户未通过验证时（如未输入完整QQ号），显示错误页面：
- 锁图标 + "安全验证失败"标题
- "请输入完整的10位QQ号以继续"
- "这是为了保护您的个人信息安全"
- 提供"返回重新输入"按钮

5.2 核心技术一：一次开发，多端部署

本项目通过自定义 BreakpointConstants 断点系统实现了一套代码适配多种设备屏幕：

断点划分与设备对应：

  断点      屏幕宽度(vp)       对应设备              布局特点
  XS        [0, 320)           手表                  紧凑竖排
  SM        [320, 600)         手机竖屏              标准竖排
  MD        [600, 840)         折叠屏、小平板         居中容器、竖排
  LG        [840, +∞)          平板横屏、PC/2-in-1    宽屏左右分栏

实现原理：

(1) BreakpointObserver 监听器：通过 window.on('windowSizeChange') 实时监听窗口大小变化，将像素宽度转换为 vp 单位后，根据阈值计算当前断点（XS/SM/MD/LG），变化时通知页面重新渲染。

(2) ResponsiveStyle 响应式样式：每个断点对应一套预定义的尺寸参数，包括标题字号、正文字号、按钮宽高、间距、圆角、视频高度、容器最大宽度、是否启用宽屏布局等。页面通过 this.style() 方法获取当前断点的样式对象，所有 UI 尺寸参数都从中读取，切换设备时自动适配。

(3) 视频页面分栏策略：在大屏（LG）下，isWideLayout 为 true，采用 Row 左右分栏——左侧 layoutWeight(3) 放置视频播放区，右侧 layoutWeight(2) 放置安全警告卡；在中小屏下则为竖排 Column 布局，视频在上、提示在下。

(4) 多设备资源适配：resources 目录下分别配置了 base（手机）、tablet（平板）、2in1（桌面）三套字符串资源，应用名称和描述随设备类型自动切换。

5.3 核心技术二：自由流转（跨设备迁移）

本项目利用 HarmonyOS 分布式能力实现了应用在手机与平板之间的无缝接续流转：

实现机制：

(1) ContinuationState 迁移状态管理器：作为全局静态类，维护两个关键数据——迁移数据对象（含当前页面路径 currentPage、QQ号 qqNumber、时间戳 timestamp）和迁移标记 isRestored。页面在关键操作节点（输入QQ号、进入视频页）自动调用 save() 保存状态。

(2) EntryAbility 迁移生命周期：
  - onContinue()：当用户从 Dock 栏发起流转时，系统回调此方法。应用读取 ContinuationState 中保存的状态数据，写入 wantParam 对象，由系统序列化后传输到目标设备。
  - onCreate() / onNewWant()：目标设备启动应用时，检测 launchReason 是否为 CONTINUATION。若是迁移启动，则从 want.parameters 中提取数据并恢复，同时设置 isRestored = true。
  - onWindowStageRestore()：迁移恢复不走 onWindowStageCreate，而是走此回调。读取 ContinuationState 中的 currentPage，加载正确的页面（若之前在视频页则恢复到视频页，否则加载主页）。

(3) 权限声明：module.json5 中声明 ohos.permission.DISTRIBUTED_DATASYNC 分布式数据同步权限，使用场景为 inuse（仅使用时授权）。

流转流程示意：

  设备A（手机）                       设备B（平板）
  输入QQ号
  → saveContinuationState()
  用户从 Dock 栏发起流转
  → onContinue() 序列化数据
  ─────── 系统传输 ──────────→  onCreate / onNewWant 恢复数据
                                 → onWindowStageRestore 加载页面
                                 → 页面读取 ContinuationState 恢复原状态

(4) 页面端流转感知：Index 和 VideoPage 的 aboutToAppear() 中均检查 ContinuationState.isRestored，若为 true 则恢复之前输入的 QQ 号或确认已通过验证状态，实现无缝衔接体验。

5.4 核心技术三：ArkUI 声明式 UI

- 使用 @Entry 和 @Component 装饰器定义页面组件
- @State 实现响应式状态管理，输入变化自动更新 UI
- @Builder 构建可复用的 UI 片段（如导航栏、播放控件、警告卡）
- router.pushUrl / router.back 实现页面导航
- Video 组件配合 VideoController 实现视频播放控制

七、设计意图——防骗教育新思路

本项目并非传统严肃的说教式安全教育，而是采用了"整蛊 + 教育"的创新模式：

(1) 吸引注意：伪装成当下流行的"人格测试"应用，利用用户的从众心理吸引参与。

(2) 制造反差：用户满心期待"人格分析结果"，等来的却是 Rick Astley 的《Never Gonna Give You Up》——经典互联网文化梗。

(3) 即时教育：突如其来的视频播放后，页面立刻展示安全警告信息，让防骗意识猝不及防地深入人心。

(4) 实践记忆：相比被动接收安全提示，亲身体验"被骗"的感觉更容易形成长期记忆。

核心信息：如果你连一个"测试"都在输QQ号，那真正的诈骗可能就在下一次点击。


八、总结

QQTI人格分析系统是一个基于 HarmonyOS 平台、使用 DevEco Studio 和 ArkTS 开发的防诈骗教育应用。它巧妙借用 MBTI 等流行人格测试的外壳，以 Rickroll 这一经典网络文化符号为载体，在用户输入 QQ 号后及时展示反诈提醒，达到寓教于乐、印象深刻的安全教育效果。

项目实现的亮点功能：

  - 一次开发，多端部署：通过自定义断点系统实现手机、平板、PC 多种屏幕形态的自适应布局，同一套代码覆盖从 320vp 的手机到 1200vp 的桌面大屏。

  - 自由流转：利用 HarmonyOS 分布式能力实现手机与平板间的无缝接续，输入进度和页面状态不丢失，用户可随时随地切换设备继续体验。

  - 视频播放与反诈警示：在"人格测试"的伪装下，用 Rickroll 制造反差，配合醒目的安全警告卡片，让防骗教育变得有趣而深刻。

  - 声明式 UI 开发：全 ArkTS 编写，组件化、响应式，代码结构清晰，易于维护和扩展。

该项目是技术实践与社会价值相结合的一次有意义的探索。
